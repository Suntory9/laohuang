# ECharts 静态旅程地图重构 — 设计规格

## 背景

当前项目是「老黄游钓中国旅程图谱」的 React + Vite 静态前端。数据由离线脚本生成到 `public/data/journey.json` 和 `public/data/videos.json`，前端读取 JSON 后展示顶部筛选、地图、时间线和视频详情。

现有地图在 `src/components/MapPanel.tsx` 中实现，优先使用高德 JS API，缺少配置时回退到自绘 SVG。参考项目 `/Users/songdc/Claude-Everything` 使用单页 ECharts + GeoJSON 的行政区可视化方式。

本次重构目标是把地图从在线导航地图改成静态行政区数据地图，同时保留「旅程故事」作为第一叙事。

## 目标

构建一个可静态部署的 ECharts 地图体验：

- 保留 React + Vite 工程结构，构建产物仍然是普通静态站点。
- 移除高德地图运行时依赖，地图由 ECharts + GeoJSON 渲染。
- 以「老黄在湖北的游钓旅程」为主线，地图先呈现路线、视频点和地点故事。
- 在市州、县市区层面提供轻量统计，包括视频数、渔获、空军、天气和买菜等信息。
- 复用现有 `TopBar`、`TimelineBar`、`DetailPanel` 的主要交互，不做整站推倒重来。

## 非目标

- 第一版不做全国地图。
- 第一版不做单文件 HTML 改造。
- 第一版不追求高德/OSM 那种道路级地图能力。
- 第一版不重写离线视频数据管线。
- 第一版不扩展复杂 BI 看板，不让统计压过旅程故事。

## 产品方向

采用「旅程故事优先，县市统计跟进」。

页面第一眼应该回答：

1. 老黄这段旅程去了哪些地方？
2. 这些视频在地图上如何串起来？
3. 点击一个地点后，这里发生过哪些故事？

统计只作为地点故事的摘要和辅助判断，例如某个县市区停留多少期、是否多次空军、常见天气和晚餐关键词。

## 页面结构

保留当前三块核心体验：

- 顶部：品牌、筛选和全局统计。
- 中央：ECharts 湖北旅程地图。
- 底部：按旅程顺序排列的视频时间线。
- 右侧：点击视频或地点后打开详情/地点故事面板。

推荐布局：

```text
┌──────────────────────────────────────────────┐
│ TopBar: 标题、筛选、总视频数、覆盖地点、渔获统计 │
├──────────────────────────────────────────────┤
│                                              │
│        ECharts 湖北地图                       │
│        市州/县市区边界 + 路线 + 视频点位        │
│                                              │
│                         ┌──────────────────┐ │
│                         │ 地点故事/统计面板 │ │
│                         └──────────────────┘ │
├──────────────────────────────────────────────┤
│ TimelineBar: 视频卡片横向时间线                │
└──────────────────────────────────────────────┘
```

地点故事面板优先展示视频列表，统计放在顶部摘要区域。

## 地图行为

地图默认显示湖北旅程总览：

- 使用湖北省行政区 GeoJSON 注册 ECharts 地图。
- 行政区颜色表示当前筛选下的视频数量。
- 视频点位使用 `scatter` 或 `effectScatter` 叠加。
- 旅程路线使用 `lines` 或自定义 polyline 系列叠加。
- 当前选中视频的点位高亮。
- 点击点位打开现有视频详情面板。
- 点击行政区打开地点故事面板，并筛出该区域相关视频。
- 底部时间线点击视频时，同步高亮地图点位和路线节点。

第一版地图层级：

1. 湖北市州总览。
2. 县市区数据先作为聚合字段进入地点面板。
3. 县市区地图下钻作为第二阶段实现，不阻塞第一版上线。

这样可以先跑通 ECharts 静态地图与现有视频体验，再处理更细的行政边界数据。

## 数据模型

现有 `VideoRecord.location` 已包含：

- `province`
- `city`
- `district`
- `poi`
- `lat`
- `lng`
- `label`
- `confidence`

新增前端派生模型，不改变原始 JSON：

```ts
interface AdminAreaStats {
  key: string;
  name: string;
  level: 'province' | 'prefecture' | 'county';
  videoCount: number;
  caughtCount: number;
  skunkedCount: number;
  shoppingCostCny: number;
  weatherTags: Array<{ label: string; count: number }>;
  theoryTags: Array<{ label: string; count: number }>;
  videos: VideoRecord[];
}
```

地点口径规则：

- `province` 继续使用原字段。
- `prefecture` 第一版从 `location.city` 派生，展示名可保留短名。
- `county` 第一版从 `location.district` 派生；为空时归入市州级「未细分地点」。
- `poi` 继续使用 `location.poi`，仅用于详情和点位标签。

如果后续数据管线要增强，再把 `prefecture/county` 写回原始数据结构；第一版先用前端聚合减少改造面。

## 组件边界

### `App.tsx`

继续负责：

- 拉取 `journey.json` 和 `videos.json`。
- 管理筛选状态。
- 管理当前选中视频。
- 管理当前选中行政区。
- 将筛选后的视频分发给地图、时间线和详情面板。

新增：

- 基于 `filteredVideos` 计算 `adminStats`。
- 地图区域点击后设置 `selectedAreaKey`。

### `MapPanel.tsx`

重写为 ECharts 地图组件。

职责：

- 初始化和销毁 ECharts 实例。
- 注册湖北 GeoJSON。
- 渲染行政区、视频点、路线。
- 将地图点击事件转换为 `onSelectVideo` 或 `onSelectArea`。
- 响应窗口尺寸变化。

不再负责：

- 加载高德 SDK。
- 高德行政区查询。
- SVG 假地图兜底。

### 新增 `lib/adminStats.ts`

职责：

- 从 `VideoRecord[]` 聚合市州/县市区统计。
- 生成 ECharts 地图 series 所需数据。
- 统一行政区名称归一化逻辑。

### 新增 `lib/echartsMap.ts`

职责：

- 封装 GeoJSON 加载。
- 封装 ECharts option 构造中的纯函数。
- 保持 `MapPanel` 不膨胀成巨大组件。

### 新增 `AreaStoryPanel.tsx`

地点级故事使用独立面板，不复用 `DetailPanel`。`DetailPanel` 继续只负责单条视频详情，避免一个组件同时承担地点聚合和视频详情两种语义。

职责：

- 展示选中行政区摘要。
- 展示该区域内视频列表。
- 点击视频时调用现有 `onSelectVideo` 打开视频详情。

## 静态资源

新增地图资源目录：

```text
public/maps/
  hubei-prefecture.json
```

第一版只要求市州级 GeoJSON。县市区 GeoJSON 后续加入：

```text
public/maps/hubei-counties.json
```

ECharts 通过 npm 依赖进入 Vite bundle，不从 CDN 加载。

## 交互细节

- 鼠标悬停行政区：显示地区名、视频数、渔获/空军摘要。
- 鼠标悬停视频点：显示视频标题、日期、地点。
- 点击行政区：选中地点故事，不直接打开单视频详情。
- 点击视频点：打开单视频详情。
- 点击时间线视频：打开单视频详情，并在地图高亮对应点。
- 筛选变化后：
  - 地图颜色按筛选结果重新计算。
  - 点位和路线只保留筛选结果。
  - 若当前选中视频不在筛选结果内，沿用现有逻辑切换到第一条匹配视频。

## 错误与空状态

- GeoJSON 加载失败：地图区域显示静态错误提示，顶部和时间线仍可用。
- 当前筛选无视频：地图显示空状态，时间线显示「无匹配视频」。
- 视频无经纬度：仍参与地点统计，但不渲染点位。
- 视频无县市区：归入对应市州的「未细分地点」。
- 行政区名匹配失败：统计仍展示在侧栏，地图不着色，并在开发环境输出警告。

## 响应式

- 桌面端：地图占主视觉，地点故事面板在右侧。
- 平板端：地点故事面板可覆盖地图右侧，宽度收窄。
- 移动端：地图在上，地点故事和时间线纵向排列；视频详情面板全屏。

## 验证

最小验证路径：

- `npm run typecheck`
- `npm run build`
- 浏览器打开构建或开发服务，确认：
  - 湖北地图可见。
  - 视频点位可见。
  - 时间线点击能高亮地图点。
  - 地图点位点击能打开现有详情。
  - 筛选会影响地图、时间线和统计。

## 实施顺序

1. 新增 ECharts 依赖和地图资源。
2. 新增地点聚合工具。
3. 重写 `MapPanel` 为 ECharts 版本。
4. 接入行政区点击与地点故事摘要。
5. 清理高德地图相关代码和文档。
6. 运行类型检查和构建验证。

## 风险

- 湖北市州 GeoJSON 的行政区名称可能与数据字段不一致，需要名称归一化。
- 当前数据中 `city` 和 `district` 的行政层级不完全规范，第一版需要兼容空值和短名。
- ECharts 地图路线不是道路级路径，只表达旅程序列，不表达真实行驶路线。
- 如果地图 GeoJSON 文件过大，首屏加载会变慢；第一版只加载湖北市州级边界降低风险。
